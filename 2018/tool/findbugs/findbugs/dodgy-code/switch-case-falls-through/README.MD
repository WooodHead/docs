# Switch statement found where fault case is missing

SVN:

	CodeFarmerClub/baseplatform/workflow/trunk/wfpt4
	
判例：

	com.yixin.wfpt.utils.date.DateUtil.java:111
	
代码：

	public static Date getEndTime(...){
		...
		switch(makeofferType){
			case 4:
				...
				break;
			case 3:
				...
				break;
			case 2:
				...
				break;
			case 1:
				...
				break;
		}
		return ...;
	}
	
描述:

	This method contains a switch statement where default case is missing. Usually you need to provide a default case.
	Because the analysis only looks at the generated bytecode, this warning can be incorrect triggered if the default case 
	is at the end of the switch statement and the switch statement doesn't contain break statements for other cases.
	
建议：

	在switch中要提供default语句，给出默认的执行条件，如果不能规划出理想的默认执行代码，请抛出异常

[why default statement](http://stackoverflow.com/questions/4649423/should-switch-statements-always-contain-a-default-clause)
	
	switch(myVar){
		case 1: ......; break;
		case 2: ......; break;
		default: throw new RuntimeException("unreachable");
	}
